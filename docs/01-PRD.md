# 代码批判 - 产品需求文档 (PRD)

## 1. 项目概述

### 1.1 产品名称
代码批判 (Code Critic)

### 1.2 产品定位
一个基于 AI 的代码风格分析工具，帮助开发者上传代码并获得结构化、可执行的改进建议。

### 1.3 目标用户
- 初级开发者：学习代码规范和最佳实践
- 中级开发者：提升代码质量和可维护性
- 代码审查者：快速发现潜在问题
- 学生：学习编程时的代码指导

## 2. 核心功能

### 2.1 代码上传
- **文件上传**：支持拖拽或点击上传代码文件
- **支持语言**：C/C++, Python, Java, JavaScript, TypeScript, Go, Rust 等
- **文件限制**：单文件最大 500KB，支持多文件批量上传
- **代码粘贴**：支持直接粘贴代码文本

### 2.2 代码分析
- **静态分析**：代码结构、复杂度分析
- **风格检查**：命名规范、缩进、注释等
- **最佳实践**：设计模式、性能优化建议
- **安全检测**：常见安全漏洞扫描
- **AI 深度分析**：基于大模型的代码逻辑评估

### 2.3 报告生成
- **评分系统**：综合评分（0-100分）
- **问题分类**：错误、警告、建议三级分类
- **代码高亮**：在报告中标注问题位置
- **修复建议**：提供具体的代码改进示例
- **导出功能**：支持 PDF、Markdown 格式导出

### 2.4 历史记录
- **分析历史**：保存用户的分析记录
- **对比功能**：对比不同版本的改进情况
- **收藏功能**：收藏重要的分析报告

## 3. 用户流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   首页      │────▶│  代码上传   │────▶│  分析中     │
│  (Landing)  │     │  (Upload)   │     │ (Analyzing) │
└─────────────┘     └─────────────┘     └──────┬──────┘
                                               │
                                               ▼
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  导出/分享  │◀────│  查看报告   │◀────│  分析完成   │
│ (Export)    │     │  (Report)   │     │  (Complete) │
└─────────────┘     └─────────────┘     └─────────────┘
```

### 3.1 详细流程

1. **首页**：展示产品价值、功能介绍、示例报告
2. **代码上传**：
   - 选择上传方式（文件/粘贴）
   - 选择编程语言（自动检测/手动选择）
   - 确认上传
3. **分析中**：
   - 显示进度条
   - 展示分析步骤（解析→静态检查→AI分析→生成报告）
   - 预计等待时间
4. **查看报告**：
   - 综合评分展示
   - 问题列表（可点击跳转）
   - 代码对比视图
   - 详细建议说明
5. **导出/分享**：
   - 导出 PDF/Markdown
   - 生成分享链接
   - 复制报告内容

## 4. 非功能性需求

### 4.1 性能要求
- 页面加载时间 < 2秒
- 代码分析时间 < 30秒（普通文件）
- 支持并发用户：100+

### 4.2 安全要求
- 代码文件安全扫描（防止恶意代码上传）
- 用户数据隔离
- 敏感信息脱敏

### 4.3 兼容性
- 支持主流浏览器（Chrome, Firefox, Safari, Edge）
- 响应式设计，支持移动端浏览
